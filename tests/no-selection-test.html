🧪 Test: Allow Engines Without Selection Feature

⚙️ Setup Instructions

  1. Open the Selection Search options page
  2. Scroll to Advanced Settings → "Allow engines without selection"
  3. Check the checkbox to enable the feature
  4. Click Save
  5. Return to this page to test

📋 Test Checklist

Use the areas below to test different scenarios:

Test 1: Inline Popup Without Selection (Option OFF)

Expected Behavior (Option Disabled)

  • ✗ Clicking in the blue area below should NOT show the popup
  • ✓ Selecting text and clicking should show the popup normally
Click anywhere in this blue area without selecting text

Test with text selection: Select some of this text and click to verify normal behavior still works.

Test 2: Inline Popup Without Selection (Option ON)

Expected Behavior (Option Enabled)

  • ✓ Clicking in the blue area should show the popup
  • ✓ ALL search engines should appear (no filtering)
  • ✓ Engines with %s will search for empty string
  • ✓ Engines without %s will navigate directly
After enabling the option, click here without selecting any text

Test 3: Context Menu (Right-Click)

Expected Behavior

  • Option OFF: Right-click menu only appears when right-clicking on selected text
  • Option ON: Right-click menu appears when clicking anywhere on the page (selection OR page context)
Right-click anywhere in this area

Also try: Right-click on selected text to verify it still works.

Test 4: Toolbar Popup

Expected Behavior

  1. Click the Selection Search icon in the toolbar (top-right corner of browser)
  2. Option OFF: Clicking a search engine without typing text should do nothing or focus the input
  3. Option ON: Clicking a search engine without typing text should execute immediately

Test 5: Different Activator Types

Test Each Activator

Go to options and test with different activators:

  • Click: Single click without selection
  • Auto: Automatic popup should appear (or button) without selection
  • Keyboard + Mouse: Ctrl+Click (or your combo) without selection
  • Double Click: Double-click always creates selection, should work normally
Test different activation methods here

Test 6: Mixed Search Engines

Setup Test Engines

Configure these test search engines in options:

  • With %s: Google → https://google.com/search?q=%s
  • Without %s: Example → https://example.com
  • Page variable only: Current URL → https://example.com?url=%PAGE_URL
  • Special COPY: Copy → COPY
  • Special Domain: Open URL → %s

Expected Results (Option ON, No Selection)

  • Google: Opens search for empty string → https://google.com/search?q=
  • Example: Opens directly → https://example.com
  • Current URL: Opens with current page URL → https://example.com?url=...
  • Copy: Should appear (copies empty string or current page context)
  • Open URL: Should appear (treats empty as URL)

Test 7: Export/Import Settings

Test Data Persistence

  1. Enable "Allow engines without selection"
  2. Go to options → Export settings
  3. Copy the exported data
  4. Disable the option
  5. Import the previously exported data
  6. Verify the option is enabled again

Test 8: Browser Sync (If Using Chrome Sync)

Test Cross-Device Sync

  1. Enable the option on Device A
  2. Wait a moment for sync
  3. Open Chrome on Device B
  4. Check that the option is enabled on Device B

🐛 Common Issues to Watch For

  • Popup appearing when it shouldn't (option OFF but shows without selection)
  • Popup NOT appearing when it should (option ON but doesn't show without selection)
  • Only some engines showing instead of all engines
  • Browser console errors (open DevTools → Console to check)
  • Option not saving/persisting after page reload
  • Context menu not appearing in page context when option is ON
