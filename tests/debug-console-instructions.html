🔍 Debug Instructions - Use Browser Console

⚠️ The buttons don't work because extensions need special permissions

Instead, we'll use the browser's Console to check values directly.

Method 1: Check in Toolbar Popup

  1. Click the Selection Search icon in your toolbar (top-right of browser)
  2. The toolbar popup will open
  3. Right-click anywhere in that popup window
  4. Select "Inspect" from the context menu
  5. DevTools will open - click the "Console" tab
  6. Type these commands one at a time and press Enter after each:
cachedOptions

👆 This shows all options. Look for allow_engines_without_selection: true

hasQuery()

👆 Should return true if option is enabled

getQuery()

👆 Shows what's in the search box (should be empty "")

Method 2: Check in Options Page

  1. Open the Selection Search options page
  2. Press F12 to open DevTools
  3. Go to Console tab
  4. Type this command:
chrome.storage.local.get('options', (data) => console.log(data.options))

👆 This shows the saved options. Look for allow_engines_without_selection: true

Method 3: Check Background Service Worker

  1. Go to chrome://extensions/
  2. Find Selection Search
  3. Click "service worker" (it's blue link text)
  4. In the DevTools that opens, go to Console tab
  5. Type:
Storage.getOptions()

👆 Shows all options including allow_engines_without_selection

What to Look For

In any of the above methods, you should see:

{ ... allow_engines_without_selection: true, ... }

If you see false or it's missing:

  • The option wasn't saved properly
  • Try enabling it again and clicking Save
  • Then reload the extension at chrome://extensions/
